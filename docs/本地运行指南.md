# 本地运行指南

## 1. 环境要求

- Windows
- Node.js、npm
- MySQL 8.0
- MySQL 服务已启动

## 2. 环境变量

项目已提供 `.env.example`。如果需要显式配置，请复制为 `.env` 并保持如下默认值：

```env
DB_HOST=localhost
DB_PORT=3306
DB_NAME=teaching_management
DB_USER=root
DB_PASSWORD=123456
DB_SSL=false
PORT=3000
SESSION_SECRET=teaching_management_secret
NODE_ENV=development
```

说明：

- 如果不创建 `.env`，项目也会使用上述默认值启动。
- 若你的数据库用户名不是 `root`，请修改 `DB_USER`。

## 3. 安装依赖

```bash
npm install
```

## 4. 初始化数据库

```bash
npm run init-db
```

初始化脚本会自动完成以下步骤：

1. 创建数据库 `teaching_management`
2. 创建所有表、外键、索引与约束
3. 写入默认测试数据

对应 SQL 文件位置：

- `sql/database.sql`
- `sql/schema.sql`
- `sql/seed.sql`

## 5. 启动系统

```bash
npm start
```

启动成功后访问：

```text
http://localhost:3000
```

## 6. 默认测试账号

- 管理员：`admin01` / `admin123`
- 教师：`t_chen` / `teacher123`
- 学生：`s_2023001` / `student123`

## 7. 推荐验证流程

1. 使用学生账号登录并尝试选课、退课
2. 使用教师账号登录并查看教学任务、录入成绩
3. 使用管理员账号登录并查看课程、开课、公告、选课数据
4. 结合数据库客户端查看 `enrollments`、`grades`、`course_sections` 等表的数据变化

## 8. 常见问题

### 数据库连接失败

- 请确认 MySQL 已启动
- 请确认账号密码与 `.env` 中配置一致
- 请确认目标端口为 `3306`

### 页面打开后没有数据

- 请确认已执行 `npm run init-db`
- 可在 MySQL 中执行 `USE teaching_management; SHOW TABLES;` 检查表是否已创建

### 端口冲突

- 修改 `.env` 中的 `PORT`
- 例如改为 `PORT=3001` 后重新执行 `npm start`
